# Project Memory

This folder is the shared memory for `latent_void`. Update it whenever the
project direction, working state, failures, datasets, HPC setup, or assumptions
change.

Use these files as the first stop after a context reset:

- `CONTEXT.md`: big-picture goal, research framing, and current architecture.
- `PLAN.md`: active implementation plan and milestone breakdown.
- `STATUS.md`: what works, what fails, blockers, and last verified commands.
- `DECISIONS.md`: decision log with reasons, so we do not re-litigate choices.
- `ZARATAN.md`: cluster sync, paths, accounts, and run commands.

Rule of thumb: if forgetting a detail would waste an hour later, write it here.
